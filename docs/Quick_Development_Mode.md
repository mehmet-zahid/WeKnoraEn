# Quick Development Mode Guide

Solves the problem in the development workflow where every modification to `app` (backend) or `frontend` (frontend) code requires rebuilding Docker images, enabling hot updates for these two modules.

## ğŸš€ Usage

### Method 1: Using Make Commands (Recommended)

```bash
# Terminal 1: Start infrastructure
make dev-start

# Terminal 2: Start backend
make dev-app

# Terminal 3: Start frontend
make dev-frontend
```

### Method 2: Using Development Scripts

```bash
# Terminal 1
./scripts/dev.sh start

# Terminal 2
./scripts/dev.sh app

# Terminal 3
./scripts/dev.sh frontend
```

### Method 3: One-Click Startup (Interactive)

```bash
./scripts/quick-dev.sh
```

### Using Air for Backend Hot Reload

After installing Air, backend code modifications will automatically recompile and restart:

```bash
# Install Air
go install github.com/cosmtrek/air@latest

# Ensure it's in PATH
export PATH=$PATH:$(go env GOPATH)/bin

# Start with Air (auto-detected)
make dev-app
```
